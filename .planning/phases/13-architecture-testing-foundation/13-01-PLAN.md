<task type="auto">
  <name>Configure TypeScript strict mode</name>
  <files>
    tsconfig.json
  </files>
  <action>
    Edit tsconfig.json to ensure strict mode is enabled with the following compiler options:

    ```json
    {
      "compilerOptions": {
        "target": "ES2020",
        "useDefineForClassFields": true,
        "module": "ESNext",
        "lib": ["ES2020", "DOM", "DOM.Iterable"],
        "skipLibCheck": true,

        /* Bundler mode */
        "moduleResolution": "bundler",
        "allowImportingTsExtensions": true,
        "resolveJsonModule": true,
        "isolatedModules": true,
        "noEmit": true,

        /* Linting - STRICT MODE */
        "strict": true,
        "noUnusedLocals": true,
        "noUnusedParameters": true,
        "noFallthroughCasesInSwitch": true,
        "noImplicitReturns": true,

        /* Path aliases (for later plans) */
        "baseUrl": ".",
        "paths": {
          "@/core/*": ["./src/core/*"],
          "@/components/*": ["./src/components/*"],
          "@/*": ["./src/*"]
        },

        /* Vite client types */
        "types": ["vite/client"]
      },
      "include": ["src"],
      "references": [{ "path": "./tsconfig.node.json" }]
    }
    ```

    Ensure "strict": true is set. The Vite template typically includes this, but verify it's present.

    DO NOT disable or relax any strict mode options.

  </action>
  <verify>
    ```bash
    grep -E '"strict":|"noUnusedLocals":|"noUnusedParameters":' tsconfig.json
    ```
    Should show all three options set to true.
  </verify>
  <done>
    TypeScript strict mode enabled with noUnusedLocals, noUnusedParameters, noFallthroughCasesInSwitch, and noImplicitReturns all set to true.
  </done>
</task>
